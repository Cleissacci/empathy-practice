Build process — prompt to product

How the Empathy Course
came to be

The journey from the first ask to the finished, shareable course — and the work behind each step.

This flow is reconstructed from the project's artifacts and design notes — it traces the actual order in which the work was built, from grounding the brand through to the offline-ready course exports. Each phase pairs the ask with how it was addressed and what shipped.

1

Foundations

Ground the brand

The ask

"Turn the Lumina Psychology brand style guide into a design system we can build everything else on."

How it was addressed

  1. Read the authored brand style guide as the single source of truth.
  2. Extracted color, type, spacing, and elevation into design tokens.
  3. Codified the voice (warm, grounded, clear) and visual rules — ivory base, no pure white, no emoji.
  4. Wrote one import-only entry point so any file links a single stylesheet.

What shipped

styles.css tokens/colors · type · spacing · elevation readme.md · SKILL.md logo + breath-wave assets
2

Component library

Build the reusable parts

The ask

"Make on-brand, reusable UI components so screens stay consistent."

How it was addressed

  1. Authored primitives across core, forms, feedback, and brand families.
  2. Compiled them into one bundle exposed on a shared namespace.
  3. Wrote foundation specimen cards so the system is browsable in the Design System tab.

What shipped

Button · IconButton · Badge · Avatar Card · StatCard Input · Textarea · Select · Checkbox · Radio · Toggle Alert · ProgressBar Logo · BreathWave
3

Learner app

Recreate the course platform

The ask

"Recreate the Empathy Course learner experience as a working app."

How it was addressed

  1. Mapped the core flow: sign in → dashboard → module → reflection.
  2. Composed it from the component library plus a curated Lucide icon set and course data.
  3. Built the surrounding chrome — sidebar navigation and per-screen layouts.

What shipped

ui_kits/empathy-course/ LoginScreen · DashboardScreen ModuleScreen · Sidebar
4

Course module

Author the self-paced player

The ask

"Build a standalone, linear e-learning module — an Articulate/Rise-style storyboard."

How it was addressed

  1. Storyboarded a 31-slide arc: cover → objectives → agenda → five modules → knowledge check → results → summary → completion.
  2. Built the player engine — linear Back/Continue nav, arrow keys, a top progress bar and slide counter.
  3. Persisted all state (position, answers, reflections) to localStorage so progress survives a refresh.

What shipped

ui_kits/empathy-elearning/ index — player engine course-data.js · slides.jsx
5

Interactions & assessment

Make every module active

The ask

"Make learners do the work — interactive activities, and a real check on understanding."

How it was addressed

  1. Built tabs, learn-more hotspots (with narration), and drag-and-drop with a tap-to-assign fallback.
  2. Replaced open journaling with applied scenarios that test each module's empathy "device."
  3. Gated Continue until each activity is done; added a scored 5-question check with an 80% pass and a try-again reset.
  4. Generated a completion certificate to PNG on a canvas — works offline — and a facilitator answer key.

What shipped

interactions.jsx · roleplay.jsx gating + scored quiz Facilitator Answer Key.html certificate (PNG)
6

Delivery

Package it to share

The ask

"Make it easy to hand off — something that opens anywhere, even offline."

How it was addressed

  1. Bundled the course and the learner app into single-file HTML exports.
  2. Inlined every asset so each file opens standalone with no server or network.
  3. Kept the brand splash and graceful no-JavaScript fallback on each export.

What shipped

The Empathy Practice.html Lumina Empathy Course.html
